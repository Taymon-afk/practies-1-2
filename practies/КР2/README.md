В рамках практических занятий с 7 по 11 была разработана полноценная система аутентификации и авторизации с управлением товарами и пользователями. Проект включает в себя бэкенд на Node.js и фронтенд на React.

В седьмой практике была реализована базовая аутентификация с хешированием паролей с помощью bcrypt. Созданы маршруты для регистрации и входа пользователей, а также полный CRUD для товаров. Пароли хранятся в базе в захешированном виде, при входе происходит сравнение хешей. Восьмая практика добавила JWT-токены. Теперь при успешном входе сервер выдает access-токен, который клиент сохраняет и отправляет с каждым запросом. Был создан middleware для проверки токенов, а также защищенный маршрут /api/auth/me для получения информации о текущем пользователе. Девятая практика расширила систему refresh-токенами. Access-токен живет 15 минут, refresh-токен — 7 дней. Когда access-токен истекает, клиент отправляет refresh-токен на /api/auth/refresh и получает новую пару токенов. При этом происходит ротация — старый refresh-токен удаляется, новый сохраняется. На фронтенде настроен перехватчик Axios, который автоматически обновляет токены при получении ошибки 401.

Десятая практика добавила фронтенд на React. Были созданы страницы регистрации и входа, форма для создания и редактирования товаров, список товаров с возможностью удаления. Все запросы к API автоматически добавляют access-токен в заголовок Authorization. При истечении токена перехватчик сам отправляет запрос на обновление и повторяет исходный запрос. Одиннадцатая практика внедрила систему ролей RBAC. Теперь в системе три роли: пользователь (только просмотр товаров), продавец (создание и редактирование товаров) и администратор (все права продавца плюс управление пользователями). Первый зарегистрированный пользователь автоматически становится администратором. Были добавлены middleware для проверки ролей и новые маршруты для управления пользователями. На фронтенде интерфейс адаптируется под роль пользователя — администратор видит раздел "Пользователи" и может менять роли, блокировать пользователей, удалять любые товары.

Проект состоит из двух папок: server и client. Для запуска нужно открыть два терминала, в первом запустить бэкенд командой npm run dev в папке server, во втором — фронтенд командой npm run dev в папке client. После этого приложение доступно по адресу http://localhost:5173. Первым делом нужно зарегистрироваться — первый пользователь получит роль администратора. Затем можно войти и начать работу. В зависимости от роли будут доступны разные функции. При истечении access-токена через 15 минут обновление происходит автоматически, пользователь не замечает этого процесса.
